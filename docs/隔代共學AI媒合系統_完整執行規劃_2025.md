- **社交網路**：顯著提升參與者的社區信任感

## 🤖 AI配對演算法詳細設計

### 演算法選擇：協同過濾（Collaborative Filtering）

**為何選擇協同過濾？**

| 演算法類型 | 優勢 | 劣勢 | 適用性 |
|-----------|------|------|--------|
| **協同過濾** | ✅ 考量歷史互動評價<br>✅ 可發現隱藏偏好<br>✅ 持續優化 | ⚠️ 冷啟動問題 | ⭐⭐⭐⭐⭐ |
| 內容基礎 | ✅ 無冷啟動問題 | ❌ 無法發現新偏好<br>❌ 需大量特徵工程 | ⭐⭐⭐ |
| 深度學習 | ✅ 最精準 | ❌ 需海量數據<br>❌ 黑箱難解釋 | ⭐⭐（初期不適用） |

**決策**：初期使用**混合式協同過濾**（User-based + Item-based），累積數據後升級深度學習

### 配對流程

```
步驟1: 特徵提取
├─ 志工特徵向量化（技能、經驗、個性、時間）
├─ 兒童特徵向量化（需求、學習風格、時段）
└─ 計算基礎相似度（Cosine Similarity）

步驟2: 協同過濾
├─ User-based CF: 找出相似兒童的成功配對
├─ Item-based CF: 找出相似志工的高評價模式
└─ 綜合評分 = 0.6 × User-based + 0.4 × Item-based

步驟3: 約束條件過濾
├─ 時間可行性（志工與兒童時段重疊）
├─ 物理能力（志工體力 vs 活動強度）
├─ 家長偏好（語言、文化背景）
└─ 志工意願（年齡段偏好、班級大小）

步驟4: 排序與推薦
├─ 產生Top 5配對候選
├─ 附上配對理由說明（可解釋性）
└─ 家長與志工雙向確認
```

### Python 實現示意（使用Surprise庫）

```python
from surprise import Dataset, Reader, KNNBasic
from surprise.model_selection import cross_validate
import pandas as pd

# 步驟1: 準備數據（歷史互動評分）
data = {
    'volunteer_id': ['V001', 'V001', 'V002', 'V003', ...],
    'child_id': ['C001', 'C002', 'C001', 'C003', ...],
    'rating': [5, 4, 5, 3, ...]  # 互動後的滿意度評分 1-5
}
df = pd.DataFrame(data)

# 步驟2: 載入數據到Surprise框架
reader = Reader(rating_scale=(1, 5))
dataset = Dataset.load_from_df(df[['volunteer_id', 'child_id', 'rating']], reader)

# 步驟3: 訓練協同過濾模型（User-based）
sim_options = {
    'name': 'cosine',  # 相似度計算方法
    'user_based': True  # User-based CF
}
algo = KNNBasic(sim_options=sim_options)
algo.fit(dataset.build_full_trainset())

# 步驟4: 預測新配對的匹配度
prediction = algo.predict('V001', 'C005')
print(f"預測評分: {prediction.est:.2f}")

# 步驟5: 為特定兒童推薦志工
def get_top_volunteers(child_id, n=5):
    """為特定兒童推薦Top N志工"""
    all_volunteers = df['volunteer_id'].unique()
    predictions = [algo.predict(v_id, child_id) for v_id in all_volunteers]
    predictions.sort(key=lambda x: x.est, reverse=True)
    return predictions[:n]

recommendations = get_top_volunteers('C005', n=5)
for rec in recommendations:
    print(f"志工 {rec.uid}: 預測滿意度 {rec.est:.2f}")
```

### 配對演算法增強（解決冷啟動）

**問題**：新加入的志工或兒童沒有歷史評分，協同過濾無法運作

**解決方案：內容基礎回退機制**

```python
def hybrid_recommendation(child_id, n=5):
    """混合式推薦：協同過濾 + 內容基礎"""

    # 檢查該兒童的歷史互動數量
    interaction_count = df[df['child_id'] == child_id].shape[0]

    if interaction_count >= 3:
        # 有足夠歷史數據，使用協同過濾
        return collaborative_filtering(child_id, n)
    else:
        # 冷啟動，使用內容基礎（技能匹配）
        return content_based_matching(child_id, n)

def content_based_matching(child_id, n=5):
    """基於技能與需求的精確匹配"""
    child_needs = get_child_needs(child_id)  # 取得兒童需求

    # 計算每個志工與兒童需求的匹配分數
    scores = []
    for volunteer in all_volunteers:
        score = calculate_skill_match(volunteer.skills, child_needs)
        scores.append((volunteer.id, score))

    # 排序並返回Top N
    scores.sort(key=lambda x: x[1], reverse=True)
    return scores[:n]
```

### 配對可解釋性設計

```python
def generate_match_explanation(volunteer, child, score):
    """生成配對理由說明"""
    reasons = []

    # 技能匹配
    matched_skills = find_matched_skills(volunteer.skills, child.needs)
    if matched_skills:
        reasons.append(f"✅ 技能匹配：{volunteer.name}擅長{matched_skills}，正符合{child.name}的學習需求")

    # 歷史成功案例
    similar_children = find_similar_children(child)
    if similar_children:
        reasons.append(f"✅ 過往經驗：{volunteer.name}成功指導過{len(similar_children)}位類似年齡的孩子，平均滿意度4.5分")

    # 個性匹配
    if volunteer.personality_traits['patience'] >= 8 and child.learning_style['concentration_span'] == '短':
        reasons.append(f"✅ 個性適配：{volunteer.name}耐心度極高，適合專注力較短的孩子")

    # 時間便利性
    if check_time_overlap(volunteer.availability, child.availability):
        reasons.append(f"✅ 時段吻合：雙方可用時段完全重疊，無需額外調整")

    return {
        'match_score': score,
        'reasons': reasons,
        'recommendation': '強烈推薦' if score >= 4.5 else '建議嘗試'
    }
```

#### DeepFace技術規格

```python
from deepface import DeepFace
import cv2

def analyze_interaction_emotion(video_stream):
    """分析互動過程中的情緒變化"""

    # 步驟1: 從監視器擷取畫面
    frame = video_stream.read()

    # 步驟2: DeepFace情緒分析
    result = DeepFace.analyze(
        img_path=frame,
        actions=['emotion'],
        enforce_detection=False  # 允許部分遮擋
    )

    # 步驟3: 提取情緒分布
    emotions = result[0]['emotion']
    # 返回: {'angry': 0.1, 'disgust': 0.0, 'fear': 0.05,
    #        'happy': 75.3, 'sad': 2.1, 'surprise': 5.2, 'neutral': 17.3}

    return emotions

def calculate_engagement_score(emotions):
    """計算互動投入度分數（0-100）"""

    # 正向情緒權重
    positive_weight = emotions['happy'] * 1.0 + emotions['surprise'] * 0.5

    # 負向情緒扣分
    negative_weight = emotions['sad'] * 0.8 + emotions['angry'] * 1.0

    # 中性過高表示無聊
    neutral_penalty = max(0, emotions['neutral'] - 30) * 0.5

    score = positive_weight - negative_weight - neutral_penalty
    return max(0, min(100, score))
```

### 監測技術3：綜合品質評分系統

#### 多維度評分模型

```python
class InteractionQualityScorer:
    """互動品質綜合評分系統"""

    def __init__(self):
        self.weights = {
            'emotion': 0.4,      # 情緒面向（最重要）
            'participation': 0.3, # 對話參與度
            'duration': 0.2,     # 持續時間（穩定度）
            'diversity': 0.1     # 活動多樣性
        }

    def calculate_overall_score(self, session_data):
        """計算整體互動品質分數（0-100）"""

        # 1. 情緒分數（雙方平均快樂度）
        emotion_score = (
            session_data['child_happiness'] * 0.6 +
            session_data['elder_happiness'] * 0.4
        )

        # 2. 參與度分數
        participation_score = session_data['conversation_rate']

        # 3. 持續時間分數（能否完成預定時長）
        planned_duration = session_data['planned_minutes']
        actual_duration = session_data['actual_minutes']
        duration_score = min(100, actual_duration / planned_duration * 100)

        # 4. 活動多樣性（是否有不同類型互動）
        activity_types = len(session_data['activities'])  # [說故事, 畫畫, 遊戲]
        diversity_score = min(100, activity_types * 33)

        # 加權總分
        total_score = (
            emotion_score * self.weights['emotion'] +
            participation_score * self.weights['participation'] +
            duration_score * self.weights['duration'] +
            diversity_score * self.weights['diversity']
        )

        return {
            'overall_score': round(total_score, 2),
            'breakdown': {
                'emotion': emotion_score,
                'participation': participation_score,
                'duration': duration_score,
                'diversity': diversity_score
            },
            'grade': self._assign_grade(total_score)
        }

    def _assign_grade(self, score):
        """評定等級"""
        if score >= 85: return 'A+ 卓越互動'
        elif score >= 75: return 'A 優質互動'
        elif score >= 65: return 'B 良好互動'
        elif score >= 50: return 'C 普通互動，有改善空間'
        else: return 'D 需要調整配對或支持'
```

## 🔐 隱私保護與倫理設計

### 國際隱私法規分析（GDPR啟示）

#### 調研發現：瑞典學校案例的教訓

**案例背景**（2019）：
- 瑞典某學校使用人臉辨識監控學生出席
- 學校主張「學生已同意」
- **結果**：瑞典資料保護局重罰，裁定同意無效

**裁定理由**：
> 學生與學校存在**權力不對等關係**，無法自由給予同意
> 兒童在依賴關係中的「同意」不具法律效力

**對我們的啟示**：
- ❌ 不能僅依賴「家長同意」作為法律基礎
- ✅ 必須設計**最小化資料收集** + **去識別化**機制
- ✅ 提供**退出機制**且不影響服務權利

### 隱私保護設計原則

#### 原則1：資料最小化（Data Minimization）

```
❌ 不需要: 錄影存檔、人臉辨識資料庫、個人影像儲存
✅ 僅收集: 即時情緒統計、匿名化互動評分、總體品質指標
```

#### 效益3：家庭時間節省

**竹科家庭痛點**：每週花4小時接送孩子上課外活動
**解決方案**：館內一站式服務，節省交通時間

| 效益項目 | 量化指標 | 貨幣化估算 |
|---------|---------|----------|
| 時間節省 | 每家庭每週節省2小時 | 2hr × 500元時薪 × 30週 = 30,000元/年 |
| 50個家庭總效益 |  | **1,500,000元** |

#### SROI計算

```
社會投資報酬率(SROI) = 總社會效益 / 總投資成本

總社會效益 = 1,450,000(兒童) + 1,050,000(長者) + 1,500,000(家庭)
           = 4,000,000元/年

總投資成本 = 565,000(初期) + 195,000(年營運)
           = 760,000元/年

SROI = 4,000,000 / 760,000 = 5.26

結論: 每投入1元，產生5.26元的社會價值
```

**與國際標準比較**：
- 社福專案SROI中位數：3-4
- 我們的5.26 → **高於平均水準**
